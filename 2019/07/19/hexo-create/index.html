Hexo 정의

Hexo는 Node.js 기반 정적 사이트 생성기(Static site generator)의 일종이다.

많은사람들이 github blog 와 연동하여 markdown으로 문서를 작성하고, hexo의 여러가지 테마를 통해서 블로그를 운영하는 방식으로 사용한다.

지난시간 마크다운에대해서 배웠다. 잠깐 사담을하자면, 필자는 회사에서 퇴사한 선배가 작성한 회사기술문서.md 를보고 마크다운을 알게됐다.

markdown문서를 몇개 보다보니, 문서구조가 눈에 굉장히 익숙했는데 여러 블로그들이나, github에서 주로 볼수있었고 얼마가지않아 github에 readme 문서가 markdown으로 작성된다는걸 알게되었다.

필자는 개발자로 처음 개발자로써 활동할때부터 기술을 잘정리해서 문서화하고, 블로그를통해 공유해야겠다고 생각하고있었다.

생각만하고 게을러서 하지않았다..

하지만, 블로그를 제작하고 운영하기 위해서는 html, css, js, 서버, 배포, 호스팅, 도메인, 트래픽 등등… 필요한기술들이 정말많고 부담스러웠다.

그렇게 개발자블로그에 관해서 찾아보던중… 마크다운 문법을 지원하는 블로그가 많다는걸 알게되었다. 그렇게해서 여러게를 찾아봤는데

github blog와 Static site generator 를 이용하면 마크다운 배포와, 호스팅 간편하게 사용할수있게 된다는걸 알게됐다.

그리고, 도메인도 github에 개인id로 들어가기때문에 github 계정을 노출하는것또한 큰장점이라 생각한다. 대중적으로 많이사용하는 Static site generator 중 크게 3가지가 있다.

  • Hexo - Node.js 기반
  • hugo GO 기반
  • jekyll Rubi 기반

이 셋중 Hexo를 채택한 이유는 Node.js는 현재 웹개발자인 나에게 익숙한 기술이고 개발경험이 있기떄문에 채택했다.

이제 Hexo를통해 마크다운 문서를 쉽게 github blog에 작성하고 배포하는법을 알려주겠다.

Markdown과 Hexo 두개만 있으면 정말쉽게 블로그를 운영할수있다
